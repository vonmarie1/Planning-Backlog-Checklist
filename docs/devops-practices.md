# Software Management, DevOps & Cloud Practices (Educational Project)

## 1. Version Control (Git & GitHub)
This project uses Git and GitHub to demonstrate version control and collaborative software development.

- All changes are tracked using commits
- GitHub repository stores project history
- Supports structured development workflow

---

## 2. Continuous Integration (CI)
GitHub Actions is used to demonstrate Continuous Integration concepts.

- Automated tests run on every push
- Ensures code consistency
- Helps detect errors early
- Used for educational demonstration of CI pipelines

---

## 3. Cloud Deployment (Render)
The project is deployed using Render, a cloud hosting platform.

- Converts local Node.js app into a live web service
- Demonstrates real-world deployment workflow
- Allows public access to the application via URL
- Shows transition from local development to cloud hosting

---

## 4. Logging and Monitoring
A simple logging system is implemented for monitoring purposes.

- Logs stored in `logs/app.log`
- Tracks:
  - Server startup
  - Requests
  - Errors
  - User connections (if applicable)

---

## 5. Software Management Practices
This project demonstrates:
- Proper folder structure
- Version control using Git
- Basic testing with Jest
- Documentation practices
- Cloud deployment (Render)
- CI pipeline integration

---

## 6. Summary
This project simulates real-world software development practices including GitHub workflow, CI, logging, and cloud deployment using Render, all for educational purposes.
